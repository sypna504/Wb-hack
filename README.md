# Wb-hack

Финальная версия проекта по прогнозированию временных рядов, выполненного в рамках хакатона.  
Репозиторий содержит завершённый pipeline решения: от загрузки и анализа данных до построения признаков, обучения модели, валидации и формирования итогового submission.

## Описание проекта

Проект посвящён задаче краткосрочного прогнозирования целевой переменной `target_1h` по временным данным маршрутов.  
В качестве исходных данных используются временные метки, `route_id`, статусные признаки и производные временные характеристики.

В рамках проекта была построена полноценная baseline/финальная версия решения, включающая:
- исследование структуры и поведения данных
- анализ временных закономерностей и сезонности
- генерацию календарных, лаговых и rolling-признаков
- добавление событийных признаков маркетплейса
- обучение модели градиентного бустинга
- временную кросс-валидацию
- калибровку прогноза
- формирование итогового файла предсказаний

## Что реализовано

- **Загрузка и подготовка данных**  
  Обработка обучающего и тестового наборов в формате Parquet, проверка структуры таблиц, типов признаков и полноты данных.

- **Исследовательский анализ данных (EDA)**  
  Выполнен анализ временного поведения признаков и таргета:
  - динамика статусных признаков
  - распределение `target_1h`
  - различия между маршрутами
  - сезонность по часам и дням недели
  - корреляционный анализ числовых признаков

- **Feature Engineering**  
  Построены информативные признаки для модели:
  - временные признаки: `hour`, `weekday`, `slot_in_day`
  - лаговые признаки: `lag_1`, `lag_2`, `lag_3`, `lag_4`, `lag_8`, `lag_48`
  - rolling-статистики: `roll_mean_4`, `roll_mean_8`
  - календарные и событийные признаки:
    - `is_holiday`
    - `days_to_near_celebration`
    - признаки, связанные с ноябрьскими маркетплейс-активностями и сезонными событиями

- **Пользовательская метрика качества**  
  Реализована собственная метрика `WAPE + Relative Bias`, позволяющая учитывать не только величину ошибки, но и систематическое смещение прогноза.

- **Обучение модели**  
  Для baseline/финального решения использован `CatBoostRegressor`, обучаемый на подготовленном наборе временных и статистических признаков.

- **Time Series Cross-Validation**  
  Реализована собственная схема временной кросс-валидации для корректной оценки качества без утечки будущей информации.

- **Калибровка предсказаний**  
  После базового прогноза выполнена дополнительная калибровка предсказаний через коэффициент масштаба для снижения bias и улучшения итоговой метрики.

- **Формирование итогового результата**  
  Подготовлен submission-файл с прогнозами для тестовой выборки и сохранён файл с предсказаниями для дальнейшего анализа.

## Какая работа была проделана

В рамках проекта был выполнен полный цикл решения forecasting-задачи:

- исследована структура обучающих данных
- проанализированы временные паттерны и сезонность
- изучено поведение таргета по маршрутам, часам и слотам внутри суток
- рассчитаны лаговые зависимости между прошлыми и будущими значениями ряда
- добавлены rolling-признаки для локального сглаживания динамики
- спроектированы календарные и событийные признаки под предметную область
- собран обучающий датасет для модели
- настроено обучение `CatBoostRegressor`
- реализована кастомная временная валидация
- рассчитана авторская метрика качества
- выполнена постобработка прогноза через калибровку
- сформированы итоговые файлы предсказаний
